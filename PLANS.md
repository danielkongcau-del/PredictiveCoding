## Stage 06 Efficiency-First Charter

### Objective

Open a new efficiency-first line that asks whether the validated Stage 05 scaffold can remain viable under materially lower budget and lower compute conditions.

### Why This Charter Exists

This charter is necessary because the repository now knows two things at once:

- long-budget Stage 05 training could still improve configured-step mechanism and report-only accuracy
- the closest same-family efficiency tweak inside the `1536`-epoch ceiling was almost ineffective
- the refined v3-C line became the strongest high-budget Stage 05 mechanism result
- that high-budget result is expensive enough that it should not be confused with an efficiency win
- the narrow v3-C contract-consolidation micro-family no longer offers a credible default next move

### Stage 06 Default Hypothesis Space

- objective curriculum
- optimization-contract refinement
- update scheduling
- shorter or more direct error / transport pathways
- matched-budget efficiency contracts

### Stage 06 First Probe Framing

The first Stage 06 probe should be framed as:

- objective-curriculum / optimization-contract hypothesis space
- low-budget-first
- matched-budget
- scaffold-preserving by default

The first question is:

- whether the remaining gap is now more about optimization conflict between trajectory-side supervision and semigroup consistency than about another narrow target-geometry refinement

That means:

- keep the validated Stage 05 scaffold available:
  - v3-A explicit transport-drift decomposition
  - v3-B trajectory curriculum structure
  - v3-C semigroup-consistency structure
- do not default to another continuation / midpoint / coupled / precision / scaled geometry pass
- do not default to another long-budget proof-of-existence pass

- reopening Stage 04 package-internal work
